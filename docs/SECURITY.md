# Модель угроз и безопасность

## Принцип «read-only по умолчанию»

Сервер стартует в профиле **`analytics`** (только чтение, ~26 инструментов). Запись не доступна без явного opt-in:

- `YANDEX_DIRECT_ENABLE_WRITES=true` → профиль `campaign-editor` (управление кампаниями, чтение + запись, ~36 инструментов).
- `YANDEX_DIRECT_ENABLE_FINANCE=true` → финансовые инструменты (пополнение счёта, счета, переводы). Работает только при активном write-профиле. На read-only (`analytics`) это env-переменная не даёт никакого эффекта.

Случайный или нежелательный вызов финансового инструмента невозможен без одновременной установки **обеих** переменных.

## Финансовые инструменты

Инструменты `v4account_deposit`, `v4account_invoice`, `v4account_transfer_money` дополнительно защищены:

- Они принадлежат группе `financial`, которая явно исключена из профилей `analytics` и `campaign-editor`.
- Upstream-механика `dry_run` и sandbox в `direct-cli` позволяет проверить параметры запроса без реального движения средств.
- Finance/master-токены для этих операций должны передаваться через env (`YANDEX_DIRECT_FINANCE_TOKEN`, `YANDEX_DIRECT_MASTER_TOKEN`), а не через аргументы инструментов.

## Поведение `DISABLED_*` переменных

`YANDEX_DIRECT_DISABLED_GROUPS` и `YANDEX_DIRECT_DISABLED_TOOLS` **уточняют активную поверхность**, но не расширяют её. Если активен профиль `analytics`, установка `DISABLED_GROUPS=campaigns` не добавляет инструменты — она только убирает то, что и так входило бы в набор.

Это safe-by-default: отсутствие `ENABLED_*` с одновременно выставленным `DISABLED_*` не откроет write-инструменты.

## Запинённое дерево зависимостей

`pyproject.toml` использует точно запинённые версии (`mcp==X.Y.Z`, `direct-cli==X.Y.Z`). Файл `uv.lock` фиксирует транзитивное дерево. PyPI не может незаметно обновить зависимость на пользовательской машине.

Если нужно обновить версии, используйте `scripts/update-pins.sh` и перепрогоните тесты.

## OAuth-токен

Токен авторизации хранит `direct-cli` в `~/.direct-cli/auth.json` с правами 0600. Плагин не читает этот файл напрямую — все auth-запросы делегируются `direct auth status --format json`.

- Не передавайте токен через аргументы инструментов или переменные окружения, если не понимаете рисков.
- Не используйте `auth_setup(code=...)` на общих машинах — код авторизации будет виден в `ps` и системных логах.
- У каждого партнёра или пользователя — свой Яндекс-аккаунт. Общие токены не используйте.

## Секреты в коде и коммитах

- Не коммитьте `.env`, `.env.test`, токены или ключи.
- Pre-commit хук запускает `gitleaks` для обнаружения утечек.
- Кассеты в `tests/recordings/` проходят через `python -m tests.sanitize` (маскировка токенов и анонимизация коммерческих данных) перед коммитом.

## Известное ограничение / v0.2

Встроенный OAuth-app принадлежит upstream-проекту `direct-cli`. Это значит, что экран согласия Яндекса показывает его бренд. Возможность подключить собственный OAuth-app (форк `direct-cli` с вашим `client_id` / `client_secret`) запланирована на версию v0.2.
