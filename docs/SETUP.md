# Настройка доступа к API Яндекс.Директа

## OAuth через `auth_login` (PKCE)

Авторизация выполняется через пакет `direct-cli` (бинарь `direct`). Плагин не хранит токены самостоятельно.

После установки и запуска MCP-сервера вызовите инструмент:

```
mcp__yandex_direct__auth_login()
```

Сервер напечатает URL для авторизации. Откройте его в браузере и войдите под **своим** Яндекс-аккаунтом. Токен сохраняется локально пакетом `direct-cli` в файл `~/.direct-cli/auth.json` с правами 0600.

Каждый пользователь или партнёр авторизуется в своём аккаунте — общие токены не используются.

## Sandbox и production

> **Уточнить в актуальной документации Яндекса:** точное имя scope, порядок одобрения доступа к production API и наличие sandbox-флага могут измениться. Ниже приведены общие ориентиры, актуальные на момент написания.

### Sandbox

Тестовая среда Яндекс.Директа доступна без отдельной заявки — она предоставляется сразу после регистрации приложения в [oauth.yandex.ru](https://oauth.yandex.ru). Используйте sandbox для разработки и тестирования, прежде чем переходить к production.

Если `direct-cli` поддерживает sandbox-режим (флаг или профиль), используйте его для изолированных тестов. Уточните в документации пакета `direct-cli` актуальные параметры.

### Production

Для работы с реальными кампаниями необходим доступ к production API:

1. Зарегистрируйте OAuth-приложение на [oauth.yandex.ru](https://oauth.yandex.ru) со scope `direct:api`.
2. Подайте заявку на доступ к API Директа: [direct.yandex.ru](https://direct.yandex.ru) → Инструменты → API → Мои заявки. Укажите Client ID приложения из шага 1.
3. Дождитесь подтверждения заявки. Без одобренной заявки все запросы вернут ошибку `incomplete_registration` (код 58).

> **Примечание:** необходимость отдельной заявки на production API и её точный процесс уточняйте в [актуальной документации Яндекс.Директа](https://yandex.ru/dev/direct/). Информация выше соответствует общепринятой практике, но детали могут измениться.

## Troubleshooting

### 401 Unauthorized

Токен битый или просрочен. Повторите авторизацию:

```
mcp__yandex_direct__auth_login()
```

Или проверьте статус:

```
mcp__yandex_direct__auth_status()
```

### 403 Forbidden

Нет доступа к API или недостаточно прав scope. Проверьте:
- Заявка на production API одобрена (см. выше).
- В OAuth-приложении указан scope `direct:api`.
- Аккаунт имеет права на нужные операции.

### `incomplete_registration` (код 58)

Заявка на доступ к API не подана или ещё не одобрена. Подайте заявку через интерфейс Директа (см. раздел Production выше).

### `cli_not_found`

Бинарь `direct` не найден. Убедитесь, что `direct-cli` установлен:

```bash
pip install direct-cli
direct --version
```

## Собственный OAuth-app (v0.2)

Использование встроенного OAuth-приложения `direct-cli` означает, что экран согласия показывает его бренд. Возможность подключить собственный OAuth-app (форк `direct-cli` с вашим `client_id` / `client_secret`) запланирована на версию v0.2.
